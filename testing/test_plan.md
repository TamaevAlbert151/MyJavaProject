# Test Plan — MyJavaProject (Task Manager)

## 1. Описание проекта
Task Manager — консольное Java-приложение, позволяющее:
- добавлять задачи;
- просматривать список задач.

Структура кода:
- `App.java` — основной класс запуска;
- `TaskManager.java` — логика управления задачами.

## 2. Цели тестирования
Проверить:
- корректность добавления задач;
- корректность хранения задач;
- отображение списка задач;
- корректность работы консольного меню (если есть);
- стабильность работы приложения без ошибок.

## 3. Объекты тестирования
### Класс **TaskManager**
Методы (предполагаемые):
- `addTask(String task)`
- `getTasks()`
- `toString()` — при отображении.

### Класс **App**
- запуск приложения;
- вывод приветственного сообщения;
- вывод списка задач.

## 4. Типы тестирования
- **Unit-тестирование**: проверка методов класса TaskManager.
- **Integration testing**: взаимодействие App → TaskManager.
- **Manual testing**: запуск через консоль.

## 5. Тестовые случаи

### 5.1 Unit Test Cases — TaskManager
| № | Метод | Входные данные | Ожидаемый результат |
|---|--------|----------------|----------------------|
| 1 | addTask | "Купить хлеб" | задача добавлена, список содержит 1 элемент |
| 2 | addTask | "" | задача не добавляется / игнорируется (если предусмотрено) |
| 3 | getTasks | — | возвращает список строк |
| 4 | getTasks | после 3 добавлений | список размером 3 |
| 5 | toString | список из 2 задач | корректное текстовое представление |

### 5.2 Integration/Manual Tests — App
| № | Сценарий | Действия пользователя | Ожидаемый результат |
|---|-----------|-----------------------|----------------------|
| 1 | Запуск | `java -cp out com.example.App` | отображается приветствие |
| 2 | Добавление задач | пользователь вводит задачи | задачи сохраняются |
| 3 | Просмотр списка | команда «показать список» | задачи выводятся по порядку |
| 4 | Пустой список | запуск без добавления | отображается сообщение «Список задач пуст» |
| 5 | Множественное добавление | 5 задач подряд | все отображаются |
